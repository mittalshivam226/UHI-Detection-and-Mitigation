UHI Detection and Mitigation Simulation: Complete Technical Report (v2)

Supersedes v1. Documents all changes made in the ML pipeline v2 upgrade.

================================================================================
PART A — ORIGINAL SYSTEM (v1 Baseline — 73.1% Accuracy)
================================================================================

1. Exact Data Sources Used
- Satellite 1 (Temperature): Landsat 8 and Landsat 9 (Collection 2, Tier 1, Level 2
  'LC08/C02/T1_L2' and 'LC09/C02/T1_L2').
  - Band extracted: ST_B10 (Surface Temperature).
  - Spatial resolution: 100m for pixel sampling (GEE reducer), native 30m.
- Satellite 2 (Indices): Sentinel-2 (Harmonized Surface Reflectance
  'COPERNICUS/S2_SR_HARMONIZED').
  - Bands extracted: B8 (NIR), B4 (Red) for NDVI; B11 (SWIR), B8 (NIR) for NDBI.
  - Spatial resolution: 10m/20m native, sampled at 100m.
- Temporal Range: Multi-year compositing 2021–2024. Peak heating season
  March 1 (Month 3) through July 31 (Month 7).
- Cloud Tolerance: ≤20% cloud cover for both Landsat and Sentinel-2.

2. Exact Preprocessing Steps Performed (v1)
- Cloud Masking: CLOUD_COVER < 20 (Landsat), CLOUDY_PIXEL_PERCENTAGE < 20 (Sentinel-2).
- Compositing: Median composite over 4-year window for cloud-free mosaics.
- Normalization: StandardScaler (mean=0, std=1) on classifier inputs only.
- Urban scope: 15 samples from a 2,000m buffer (GEE sample(), seed=42).
- Rural scope: mean LST from annulus 8,000m–15,000m from city center.
- UHI Label: UHI=1 if (urban_LST - rural_mean) >= 2.0°C (fixed threshold, all cities).

3. Features Computed (v1)
- LST: (ST_B10 × 0.00341802) + 149.0 − 273.15    [Landsat]
- NDVI: (B8 − B4) / (B8 + B4)                    [Sentinel-2]
- NDBI: (B11 − B8) / (B11 + B8)                  [Sentinel-2]
- Classifier features: [lst, ndvi, ndbi]
- Regressor features:  [ndvi, ndbi, abs_lat, rural_lst_mean]

4. Model Architecture (v1)
- Classifier: RandomForestClassifier
  - n_estimators=200, max_depth=None, min_samples_leaf=2
  - class_weight="balanced", random_state=42
  - Split: 80/20 stratified (random, NOT city-grouped → leakage risk)
  - CV: 5-fold (scoring=f1)
- Regressor: RandomForestRegressor
  - n_estimators=200, max_depth=None, min_samples_leaf=2

5. v1 Evaluation Metrics (927 rows, 47.9% positive UHI rate)
- Accuracy:  0.7311
- F1-Score:  0.7282
- Precision: 0.7052
- Recall:    0.7528
- ROC-AUC:   (not recorded in v1)
- Regressor R²:   0.9072
- Regressor RMSE: 2.6384°C
- Regressor MAE:  1.7953°C

6. v1 Feature Importance (Random Forest Gini)
- Classifier: LST (0.4239), NDBI (0.3079), NDVI (0.2682)
- Regressor:  rural_lst_mean (0.8253), ndvi (0.0614), abs_lat (0.0580), ndbi (0.0552)

================================================================================
PART B — UPGRADED SYSTEM (v2 — Improved Pipeline)
================================================================================

7. Root Causes of 73% v1 Accuracy (Diagnosed)
   a) Only 3 features — insufficient decision surface for a global 60-city model.
   b) 927 rows (~15/city) — statistically underpowered for 6 climate zones.
   c) Fixed 2°C UHI threshold ignores climate-zone physics (2°C is trivial in
      a 45°C desert; meaningful in temperate London).
   d) Raw LST in classifier features + LST-derived label = partial label leakage.
   e) Random train/test split allows same city in both train and test sets —
      inflates reported accuracy (memorization vs. generalization).

8. Changes Made to collect_dataset.py (v2)
   a) NUM_SAMPLES_CITY raised from 15 → 80 (target: ~4,800 rows from 60 cities).
   b) EVI (Enhanced Vegetation Index) added from Sentinel-2 (B2, B4, B8):
         EVI = 2.5 × (NIR − RED) / (NIR + 6×RED − 7.5×BLUE + 1)
      More robust than NDVI in dense urban canopy / high-biomass areas.
   c) Elevation added from SRTM (USGS/SRTMGL1_003, band: elevation, metres).
      Corrects for topographic temperature lapse rate across diverse cities.
   d) Nighttime Lights (NTL) added from VIIRS monthly composite (2023 mean):
         Collection: NOAA/VIIRS/DNB/MONTHLY_V1/VCMSLCFG, band: avg_rad
      NTL is a direct proxy for anthropogenic heat emission (energy use, traffic).
   e) lst_delta column added = (urban LST − rural_lst_mean). This is stored in
      the CSV so the classifier uses a relative thermal anomaly, not the raw
      absolute temperature which conflates climate zones.
   f) Climate-zone-adaptive UHI thresholds replace the fixed 2°C:
         arid:          3.5°C  (desert cities have extreme ambient baselines)
         tropical:      2.5°C
         subtropical:   2.0°C
         temperate:     1.5°C  (1.5°C above rural is a significant UHI signal)
         mediterranean: 2.0°C
         monsoon:       2.5°C
   g) Output CSV now includes columns:
      lat, lon, lst, ndvi, ndbi, evi, elevation, ntl, lst_delta,
      city, country, climate_zone, zone_type, rural_lst_mean, uhi_label

9. Changes Made to train_model.py (v2)
   a) Classifier algorithm upgraded: RandomForest → XGBoost (XGBClassifier).
      XGBoost provides better gradient-boosted trees, handles feature interactions,
      and consistently outperforms RF by 2–5% on structured tabular data.
      Fallback to RandomForestClassifier if XGBoost not installed.
   b) Hyperparameter optimization via RandomizedSearchCV (n_iter=30, cv=5):
      Search space: n_estimators, max_depth, learning_rate, subsample,
      colsample_bytree, min_child_weight, gamma, scale_pos_weight.
   c) GroupKFold by city for both training split and cross-validation.
      No city can appear in both train and test — more honest generalization estimate.
   d) Classifier features (v2): [lst_delta, ndvi, ndbi, evi, elevation, ntl]
      lst_delta replaces raw lst to remove label leakage.
   e) Regressor features (v2): [ndvi, ndbi, evi, elevation, ntl, abs_lat, rural_lst_mean]
   f) Regressor n_estimators raised from 200 → 300.
   g) feature_importance.json now also stores classifier_metrics and regressor_metrics
      dictionaries (accuracy, f1, precision, recall, roc_auc, cv_f1, rmse, r2, etc.).

10. Changes Made to ml_service.py (v2)
    a) predict_uhi() extended: accepts evi, elevation, ntl, rural_lst_mean.
    b) predict_temperature() extended: accepts evi, elevation, ntl.
    c) simulate_mitigation() extended: accepts evi, elevation, ntl.
    d) Feature-version auto-detection: probes clf.n_features_in_ at load time.
       If n_features >= 6 → uses v2 vector; if n_features == 3 → uses v1 vector.
       This ensures backward compatibility with old .pkl models.
    e) Auxiliary feature defaults: evi ≈ 0.9×ndvi, elevation = 0, ntl = 0
       when not provided by caller (graceful degradation).
    f) get_status() now exposes model_version, clf_n_features, clf_feature_names.

14. SHAP Implementation
    - v1: Did not generate SHAP values. Used sklearn Gini importance only.
    - v2: XGBoost provides get_booster().get_score(importance_type='gain') which
      is equivalent to split-gain importance (more reliable than Gini for boosted trees).
      These are stored in feature_importance.json as classifier_importance.
