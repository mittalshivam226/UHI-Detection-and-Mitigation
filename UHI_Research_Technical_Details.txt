UHI Detection and Mitigation Simulation: Complete Technical Report (v2.1)

Supersedes all previous technical reports. Documents all changes made in the ML pipeline v2 upgrade, including the v2.1 Explainable AI (XAI) engine integration.

================================================================================
PART A — DIAGNOSIS OF V1 ARCHITECTURE (The 73.1% Accuracy Baseline)
================================================================================

1. Root Causes of 73% v1 Accuracy (Diagnosed)
   a) Insufficient Feature Space: Only 3 features (LST, NDVI, NDBI) were used. This failed to map global climate nuances.
   b) Statistically Underpowered: Only 927 rows (~15/city) were gathered across 6 climate zones.
   c) Physics-Blind Thresholds: A fixed 2°C UHI threshold ignored climate-zone baselines.
   d) Label Leakage: Raw LST was used in classifier features while simultaneously deriving the UHI label.
   e) Validation Leakage: Random train/test split allowed the same city in both train and test sets, artificially inflating generalization accuracy via memorization.

================================================================================
PART B — UPGRADED SYSTEM (v2.1 XAI Pipeline)
================================================================================

2. Data Collection Engine Upgrades (`collect_dataset.py`)
   a) Scale: Samples raised from 15 → 80 points per city (Yield: 4,882 nodes across 60 global cities).
   b) EVI (Enhanced Vegetation Index): Sentinel-2 (B2, B4, B8) added to penetrate dense urban canopies.
   c) Topography (Elevation): NASA SRTM added to correct for altitudinal temperature lapse rates.
   d) Anthropogenic Heat (NTL): NOAA VIIRS Nighttime Lights added as a proxy for human energy emission.
   e) LST Delta: `lst_delta` = (urban_LST − rural_lst_mean). Replaces absolute LST to completely isolate the urban anomaly signal from geographical climate baselines.

3. Core Model Architecture Upgrades (`train_model.py`)
   a) Classifier Algorithm: Random Forest → XGBoost `XGBClassifier`. Handles dense geospatial interactions with extremely high accuracy.
   b) Hyperparameter Tuning: `RandomizedSearchCV` (n_iter=30) across 8 dimensions (learning_rate, max_depth, gamma, etc.).
   c) Spatial Cross-Validation: Implemented `GroupKFold` grouped by city. Guarantees the model cannot memorize a city's thermal signature.
   d) Regressor Algorithm: `RandomForestRegressor` with n_estimators bumped from 200 → 300 to smooth out counterfactual variance.

4. Explainable AI (XAI) Integration (`ml_service.py` v2.1)
   a) The system no longer relies solely on global Gini importance.
   b) SHAP (SHapley Additive exPlanations): Integrated `shap.TreeExplainer` directly into the FastAPI response payload.
   c) Every inference dynamically calculates the local log-odds contribution (SHAP values) of all 7 variables, pushing them to the Frontend for real-time waterfall matrix rendering.
