--------------------------------------------------------------------------------
1. SYSTEM OVERVIEW
--------------------------------------------------------------------------------

The UHIS is a bleeding-edge, web-based control room designed to detect, analyze, 
and mitigate Urban Heat Island (UHI) effects globally. By synthesizing real-time 
satellite remote sensing data with explainable artificial intelligence (XAI), 
the system provides urban planners with hyper-local thermal simulations. 

Architectural stack:
  - Google Earth Engine (GEE): Parallelized fetching for live Landsat 8/9 and Sentinel-2.
  - Machine Learning Layer: XGBoost Classifier (detection) + Random Forest Regressor.
  - Explainable AI: SHAP (SHapley Additive exPlanations) for real-time inference debugging.
  - Frontend: React + Vite with Awwwards-tier 3D Three.js data holograms and framer-motion.
  - Backend: FastAPI for high-concurrency geospatial data orchestration.

--------------------------------------------------------------------------------
2. MULTI-MODAL DATASETS
--------------------------------------------------------------------------------

To ensure robust global generalization, the platform constructs a 7-feature 
hyperspectral and topographical vector for every queried coordinate using a 
1,000-meter buffer zone.

  Source 1: Landsat 8 / 9 (USGS)
    - Thermal Infrared (Band 10) calculating Land Surface Temperature (LST).
    - Annular Baseline mapping: Computes rural_lst_mean at a 3-15km radius.

  Source 2: Sentinel-2 (ESA Copernicus)
    - High-resolution (10m) multi-spectral computation.
    - NDVI (Vegetation), NDBI (Urban Density), EVI (Enhanced Vegetation).

  Source 3: NASA SRTM (Shuttle Radar Topography Mission)
    - 30-meter resolution Digital Elevation Model (DEM) for altitudinal cooling gradients.

  Source 4: NOAA VIIRS
    - Day/Night Band monthly composites for Nighttime Lights (NTL) radiance.

  Analysis Protocol: Cloud cover ≤ 20%, 2023-2024 median composite aggregation.

--------------------------------------------------------------------------------
3. MACHINE LEARNING & DIAGNOSTICS
--------------------------------------------------------------------------------

  3.1 XGBoost Classifier (UHI Probability)
      Input variables : [lst_delta, ndvi, ndbi, evi, elevation, ntl]
      Output          : Binary hotspot classification (0 / 1) and exact probability.
      Logic           : `lst_delta` (Urban vs Rural thermal difference) isolates true 
                        heat islands from globally hot climates (e.g., deserts).

  3.2 Environmental Regressor (Random Forest)
      Input variables : [ndvi, ndbi, evi, elevation, ntl, abs(latitude), rural_lst_mean]
      Output          : Baseline predicted temperature (°C).
      Purpose         : Simulates how temperature reacts to shifting urban fabric variables 
                        independent of the sun's immediate thermal load.

  3.3 Explainable AI (SHAP Waterfall Matrix)
      The engine utilizes `shap.TreeExplainer` locally to deconstruct the exact XGBoost 
      log-odds prediction. Every classification triggers a cascading data waterfall UI, 
      revealing exactly which variables (e.g., extreme NDBI, low Elevation) pushed the 
      hotspot into critical territory.

  3.4 Feature Importance Analysis
      Based on the 4,882-node global dataset:
          1. LST_Delta  — ~81.9% (By far the strongest signal for true UHI anomalies)
          2. NDBI       — ~ 8.4% (Built-up thermal mass)
          3. Elevation  — ~ 3.8% (Topographical cooling)
          4. NDVI       — ~ 3.4% (Canopy cover)

--------------------------------------------------------------------------------
4. MODEL VALIDATION METRICS
--------------------------------------------------------------------------------

Evaluated using strict Spatial K-Fold Cross-Validation to prevent geographical leakage.

  Classifier Performance:
      - ROC-AUC      : 0.985
      - Accuracy     : 93.49%
      - F1-Score     : 0.8946 (Mean: 0.89 ± 0.015 Std. Dev)
  
  Regressor Performance:
      - R² Score     : 0.9416 (94% of temperature variance explained)
      - RMSE         : ±2.08°C
      - MAE          : ±1.41°C

--------------------------------------------------------------------------------
5. MITIGATION SIMULATION DYNAMICS
--------------------------------------------------------------------------------

Users can toggle intervention layers instantly projecting climate modifications.
The engine calculates absolute limits based on literature ranges:

  ACTION               MAX COOLING  NDVI DELTA   NDBI DELTA   SOURCE
  ------------------   -----------  -----------  ----------   --------------------
  Urban Tree Cover      4.0°C       +0.18        -0.05        Bowler et al. 2010
  Cool/Albedo Roofing   5.5°C        0.00        -0.15        Santamouris 2014
  Water Features        2.5°C       +0.10        -0.06        Taha 1997
  Green Roofs           2.5°C       +0.08        -0.10        Santamouris 2014

The real-time simulator subtracts the aggregated cooling deltas directly from the 
live GEE baseline LST, visually recalculating the final metric down to 15.0°C minimums.

--------------------------------------------------------------------------------
6. CONCLUSION
--------------------------------------------------------------------------------

The upgraded Urban Heat Intelligence System marks a definitive leap in geospatial ML. 
By expanding the core vectors to include altitudinal (Elevation) and anthropogenic 
(Nighttime Lights) contexts alongside `lst_delta`, the XGBoost pipeline achieves near-perfect 
classification boundaries. The implementation of real-time SHAP analysis and K-Fold 
variance UI monitors elevates the platform from an analytical script to a highly 
robust, production-ready climate intelligence control room.
