UHI Detection & Mitigation System - Detailed Workflow Document
Use this document to generate UML Activity Diagrams, Sequence Diagrams, and State Machine Diagrams.

==================================================
1. SYSTEM ACTORS
==================================================
- Client / End User: Urban planner or researcher via Web Browser.
- Frontend Control Room (React/Vite): Handles state, Three.js 3D renderings, Maps.
- Backend API (FastAPI): High-concurrency ML orchestrator.
- ML Service (XGBoost / Shapley): Executes ML classification and XAI logic.
- GEE Service (Google Earth Engine): External thread-pooled computation.
- Simulation Engine: Python service resolving mitigation vectors.

==================================================
2. SEQUENCE OF OPERATIONS: CORE XAI WORKFLOW
==================================================
1. User clicks a point on the interactive 3D map OR searches via Mapbox Geocoder.
2. Frontend EnginePage updates global state (`selectedCoordinate`).
3. Frontend UI displays a scanning animation overlay.
4. Frontend streams an HTTP GET request to `/api/v1/ml/analyze?lat=XX&lng=YY`.
5. Backend API Gateway receives request, parsing coordinates.
6. Backend invokes `gee_service.fetch_environmental_data(lat, lng)`:
   a. ThreadPoolExecutor spawns 5 parallel GEE threads.
   b. Fetch Landsat 8/9, Sentinel-2, SRTM 30m, and VIIRS NTL.
   c. Resolve median composites and clip all bands to 1000m buffer.
   d. Compute `lst`, `ndvi`, `ndbi`, `evi`, `elevation`, `ntl`, `rural_lst_mean`.
7. `GEEService` returns 7-feature dict back to backend router.
8. Backend passes metrics to `MLService.predict_uhi(...)`:
   a. Instantiates `shap.TreeExplainer(XGBoost)`.
   b. Runs 7-feature arrays through XGBoost for UHI severity + binary tag.
   c. Generates exact local `shap_values` log-odds mapping per feature.
   d. Runs `RandomForestRegressor` for `predicted_baseline_temp`.
9. Backend compares values against predefined environmental `causes` string sets.
10. Backend passes detected causes to `RecommendationService` for actionable logic mapping.
11. Backend packages XAI, ML indices, and 7-features into JSON response.
12. Frontend intercepts 200 OK.
13. Frontend UI (`EnginePage.jsx`) updates specific visualization components:
    - SHAP Waterfall UI (ECharts cascading bar graph)
    - K-Fold Variance Heartbeat (EKG-style line chart)
    - 3D Holographic Globe
14. System rests in "ANALYZED" state awaiting further human input.

==================================================
3. SEQUENCE OF OPERATIONS: MITIGATION SIMULATION
==================================================
1. User interacts with UI Simulation Switches (e.g. "Add Arbor Canopies").
2. User adjusts % adoption intensity sliders via Framer Motion.
3. Frontend builds an array of requested interventions.
4. Frontend executes an HTTP POST to `/api/v1/ml/simulate` containing existing `lst_celsius` and selected actions.
5. Backend maps payload to `SimulationService.simulate()`.
6. For each item in payload:
   a. Calculate cooling bounded by literature limits: `reduction = max_cooling * (slider_pct/100)`.
   b. Accumulate total thermal offset.
7. Backend asserts an absolute minimum LST floor (temp cannot fall below 15.0°C).
8. Backend returns JSON payload describing `.new_temp`, `.original_temp`, and `.reduction`.
9. Frontend animates the "Counterfactual Result" numerical HUD metric without reloading.

==================================================
4. SEQUENCE OF OPERATIONS: LIVE TELEMETRY (Background)
==================================================
1. Independent of User Interaction, Frontend component triggers `setInterval`.
2. Every 4.5 seconds, un-cached ping deployed to `/api/v1/data/ping`.
3. Backend immediately responds with HTTP 200 taking ~13ms.
4. Frontend `EnginePage.jsx` updates `latencyHistory` queue.
5. Bottom-left ECharts line graph continually updates simulating live "Control Room" active pulse.
